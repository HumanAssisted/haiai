# Cross-Language Media Verify-Parity Fixtures

Pre-signed image and markdown fixtures used by `cross_lang_media_*` tests in
Rust, Python, Node, and Go. The single set of bytes here MUST verify under
`status: "valid"` in all four SDKs — that is the contract test for media
sign/verify parity (PRD §5.5, MEDIA_SIGNING_PRD_TASKS/MEDIA_SIGNING_TASK_011).

## Layout

- `_source/source.{png,jpg,webp,md}` — unsigned input bytes
- `signed.{png,jpg,webp,md}` — JACS-signed counterparts (committed)
- `CHECKSUMS.txt` — sha256 watchdog (catches byte drift in CI)
- `SIGNER.json` — `{signer_id, algorithm, fixture_agent_dir}` so language
  tests can read the expected signer identity instead of hardcoding it

## Provenance

Source images are deterministic flat-color pixels generated by the
`image` crate (PNG, JPEG) plus a hand-built minimal RIFF container
(WebP). The markdown is plain ASCII. All four are public-domain by
construction; no third-party content is embedded.

The signing identity is the shared `fixtures/jacs-agent/` agent
(`jacs_agent_id_and_version` in its config). Signatures are produced by
JACS 0.10.0 via `LocalJacsProvider` with default `SignImageOptions` /
`SignTextOptions` — metadata-channel only (no robust LSB embedding) for
byte determinism.

## Regenerating

JACS signatures embed a timestamp (`jacsVersionDate`), so re-signing
produces different bytes. CI must NOT regenerate; that would defeat the
parity check.

To refresh locally (e.g. after a JACS schema change):

```
cargo test -p haiai --test regen_media_fixtures -- --ignored
```

The regenerator:

1. Re-derives the four `_source/*` byte sequences (deterministic).
2. Stages the `fixtures/jacs-agent/` agent in a tempdir (with the
   colon-filename mapping `prepare_jacs_fixture` uses).
3. Signs each source via `LocalJacsProvider`.
4. Writes `signed.*`, `CHECKSUMS.txt`, and `SIGNER.json`.

After regenerating, commit all five files together. The fixture agent
password is `secretpassord` (same as
`rust/hai-mcp/src/embedded_provider.rs` test harness).
